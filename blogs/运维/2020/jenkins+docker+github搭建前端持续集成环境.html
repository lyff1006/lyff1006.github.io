win10环境 jenkins+docker+github搭建前端持续集成环境

小羽 2020-07-19 jenkinsdockergithub持续集成win10

# 1.docker安装

关于docker的安装，没啥好说的，直接官网下载然后直接傻瓜式安装即可（以前有装过vm等虚拟机的童鞋记得把虚拟机卸载了，然后重启，接着开启win10的hyper-v虚拟机，否则安装会报错）。https://www.docker.com/get-started

# 2.安装jenkins

# 2.1 下载jenkinsjenkins

下载jenkins的win10版本

# 2.2 安装jenkins

这一步没啥好说的，选择好jenkins的安装路径，然后傻瓜式安装jenkins

# 2.3 进入jinkins

容器创建启动完毕后，在url地址中输入localhost:6701(你映射的端口，默认是8080，这里本人修改为6701端口了)，即可进入jenkins。jenkins第一次启动时间会比较久请耐心等待。

然后通过提示将密码复制在下方的密码框中

# 2.4 安装jenkins插件

好了，找到密码了，然后就可以进入jenkins了，咦，不对，这又是个啥？噢，原来是插件安装，我们就直接选择推荐的插件就可以了，然后又是漫长的等待。

ps：小伙伴们觉得慢的话可以使用国内源进行加速，进入配置目录（自己的jenkins安装路劲/jenkins_workplace/updates）。将所有的jenkins源替换为清华源

http://updates.jenkins-ci.org/download/plugins/

https://mirrors.tuna.tsinghua.edu.cn/jenkins/plugins/

# 2.5 设置账号密码

插件装好了，就可以进入jenkins中设置账号和密码了，然后剩下的步骤都直接默认即可。然后终于看到了jenkins的控制台了，不容易啊，铁子们~

# 2.6 配置node环境

（1）安装node插件

进入设置，manage plugin 中搜索node，然后安装，安装完后重启jenkins的docker镜像

（2）进入设置->全局设置，添加node环境

# 3.结合github中的webhook配置持续集成

# 3.1 创建github密钥

# 3.2 jenkins创建项目

# 3.2.1 新建项目

我们先来创建一个新的项目，输入项目名称后点击确定

# 3.2.2 项目持续集成配置

（1）输入github项目地址，指定分支那里是指将要构建的项目分支默认为master。然后把github hook trigger for giiscm polling（这个选项是为了后面的自动化持续集成做准备）

（2）credentials中添加github账号和密码

（3）选择node环境

（4）添加构建步骤

（5）构建测试

保存后，点击build now即开始构建，点击下方的序号可以查看我们的构建的工程

# 3.3 结合ngrok部署内网穿透服务

众所周知知道ipv4资源紧缺，运营商也一般都不会给我们配备公网ip。因此想让我们的服务可以从外网进行访问就需要部署内网穿透服务（当然也可以使用ipv6，但是目前还不算完善，很多地方ipv6的网络还不全）

1.注册ngrok账号

http://www.ngrok.cc/

2.隧道管理，选择免费服务器

3.填写相关信息，其中本地端口哪里对应的是从docker中映射出来的端口（本文2.2节）

4.下载ngrok客服端，输入隧道id，开启ngrok穿透服务

5.测试内网穿透是否成功

在浏览器中输入ngrok给我们提供的url地址，如果可以进入jenkins的页面说明我们已经成功穿透到公网了

# 3.4 结合webhook

（1）进入github相关项目的setting->webhooks

（2）添加webhook

（4）在我们的vue工程文件中添加dockerfile和nginx.conf

（5）修改项目的构建步骤，修改为通过dockerfile进行构建。然后随便在自己的项目中修改一些东西，然后推送到github上的release分支。回到jenkins中，可以发现项目已经在进行自动构建了，查看构建的结果。发现可以通过localhost:6801对刚刚构建好的项目进行访问，到此前端的自动化构建完成。
